Review tổng thể (Business logic / Code pattern / Architecture)

Business logic hiện tại

  • Auth: login/register cấp JWT + lưu session; AccessRequest cho phép user tự gửi yêu cầu xin thêm role/data-permission (duyệt sau), không trực tiếp quản lý.Tạo/cập nhật user, gán role (ROLE-url/page) và data-permission trực tiếp qua API/console (hoặc approve AccessRequest).
  • Authorization: kiểm tra URL theo bảng url_permissions, cache RAM cục bộ.
  • Data scope: mỗi user gắn một data_permission_id → join data_permissions → sets/datasets; build WHERE động (table/col/val) theo dịch vụ, hỗ trợ wildcard “*”; mặc định deny nếu không có scope khớp.
  • AccessRequest: duyệt chỉ áp dụng roles và data_permission; trường set_ids đã loại khỏi luồng duyệt (data scope đi theo data_permission).

Kiến trúc & pattern

  • Layers: controllers → services → repositories (ORM), utils cho cache/audit/idempotency.
  • ORM: hầu hết repo chuyển sang SQLAlchemy ORM; QueryBuilder dùng expression; không bật FK theo chủ đích.
  • DI: services khởi tạo qua container; engine/session cấu hình qua container; vẫn còn một số cấu hình khác đọc Config trực tiếp.
  • Caching: Redis cho permission list; authz cache nằm trong bộ nhớ process.

Rủi ro / Tech debt

  • Chưa có FK ở DB (chủ động); toàn vẹn dữ liệu phụ thuộc code và trình tự xoá.
  • AuthZ cache không invalidate khi đổi role/permission; TTL mới xóa.
  • AccessRequest chưa áp dụng set_ids; audit/log mới phần này.
  • Nhiều file xóa (person/department) đang ở trạng thái D trong git; cần dọn commit.
  • Alembic baseline no-op: cần quyết định có bật FK/migration hay giữ hiện trạng.

Hành động khuyến nghị

  • Dọn nốt text() còn lại, chuẩn hóa scope filter sang expression; bổ sung test cho QueryBuilder mới.
  • Đưa Redis/DB/client vào container/factory để dễ mock; bổ sung test integration.
  • AuthZ cache sang Redis (hoặc cache layer chung) và flush khi cập nhật role/permission.
  • Hoàn thiện logic set_ids trong AccessRequest hoặc bỏ field khỏi API.
  • Chuẩn hóa logging/error code; thêm audit cho create/update/delete role/permission.
