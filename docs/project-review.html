Đánh giá dịch vụ UAA

Phạm vi

Đã hoàn thành rà soát toàn bộ module service trong uaa/services. Bao gồm business intent, logic chính, pattern kiến trúc và tech stack.

1. Tóm tắt

  • Tầng service tách biệt với controllers và repository.
  • Repository pattern (User/Role/Permission/Set/AccessRequest).
  • Dependency injection qua uaa/services/container.py với singleton factory.
  • Auth flow: JWT, bcrypt (tùy), session Flask.
  • Workflow access request (submit/approve/reject/cancel/apply).

3. Tech stack

  • Python 3 + Flask
  • SQLAlchemy ORM + raw SQL
  • JWT (pyjwt)
  • bcrypt (nếu có)
  • Redis cache
  • RBAC: role/permission/url

4. Điều cần cải thiện

  • Giảm trùng lặp: AuthService/RegistrationService có nhiều code giống nhau.
  • Security: _ids_from_codes dùng f-string table name chưa validate; PasswordService fallback plaintext là nguy cơ.
  • Authorization cache di động: từ bộ nhớ process sang Redis cluster nếu scale.
  • AccessRequest create không dùng requested_set_ids (có trong tham số nhưng chưa xử lý).
  • Không dùng bare except, nên rõ exception và trả lỗi chuẩn.

5. Kết luận

Hệ thống đã hoàn chỉnh logic user+role+permission+access request. Kiến trúc rõ, dễ mở rộng và unit test. Cần củng cố bảo mật, tái cấu trúc để DRY và triển khai cache phân tán.
